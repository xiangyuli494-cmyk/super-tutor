## 第 1 章 · 项目概述

### 1.1 产品愿景

超级私教是一个**基于三 AI 角色协作的认知孪生驱动深度学习辅助系统**。用户上传 PDF 教材后，三个 AI 角色（Tutor / Assistant / Evaluator）协作完成「资料解析 → 自适应出题 → 智能排期」的完整学习闭环。

### 1.2 一句话定位

> 不是「问答机器人」——是持续追踪你每个知识点掌握度的**认知孪生私教**。

### 1.3 与竞品的核心差异

| 维度 | 扣子 / ChatGPT | 超级私教（本系统） |
|------|---------------|-------------------|
| 交互模式 | 单轮 / 多轮对话 | 持续性学习闭环 |
| 知识管理 | 每次对话独立 | PDF 上传后持久化向量索引 |
| 题目生成 | 一次性批量输出 | 基于知识库**分题型出题**（6 种题型） |
| 学习追踪 | 无 | 每个知识点的**认知孪生**（EMA 掌握度 + SM-2 排期） |
| 错误诊断 | 判对错 + 给答案 | **迷思概念分类**（7 种错误类别） + 苏格拉底式引导提示 |
| 架构 | 单一模型 | 三 AI 角色协作流水线（5 阶段状态机） |

### 1.4 目标用户

- **大学生**：专业课教材 PDF 多，备考压力大，需要系统化复习工具
- **考研 / 考公人群**：长期备考（3-12 个月），SM-2 排期天然适配
- **自学编程 / 技术人群**：大量技术文档需要结构化理解和测验

### 1.5 用户画像

| 用户 | 典型场景 | 痛点 | 超私如何解决 |
|------|---------|------|------------|
| 大学生 | 期末周 5 门课，3 本厚教材 | "来不及看，不知道从哪开始" | 上传 PDF → 自动出题 → 专攻薄弱点 |
| 考研党 | 半年备考，数学+专业课 | "复习过的忘了，没复习的不知道" | SM-2 排期自动提醒，精确到天 |
| 自学者 | 想系统学一门新技术 | "资源太多，不知道重点在哪" | 任意文档上传，自动提炼知识点 |

**用户故事：**
> 作为一名学生，我希望系统根据我上传的教材自动生成测验题，题目难度有层次递进，且每题有详细解析。

- 流水线：`orchestrator_phases.py` → `_quiz_gen_phase()`（Assistant 角色）

- 角色 Prompt：`prompts/assistant.md`（含 6 题型 few-shot 示例）
- 数据库：`questions` 表

**6 种题型分布策略：**

| 题型 | type 值 | 占比 | 前端渲染 |
|------|---------|:---:|---------|
| 选择题 | `multiple_choice` | ~40% | ✅ 单选按钮 |
| 判断题 | `true_false` | ~15% | ❌ 退化文本框 |
| 填空题 | `fill_in_blank` | ~15% | ❌ 退化文本框 |
| 简答题 | `short_answer` | ~15% | ⚠️ 文本框（可用） |
| 匹配题 | `matching` | ~10% | ❌ 退化文本框 |
| 论述题 | `essay` | ~5% | ⚠️ 文本框（可用） |

**验收标准：**

| ID | 标准 | 量化指标 | 状态 |
|----|------|---------|:---:|
| AC2.1 | 题干可理解率 | ≥ 95% | ✅ |
| AC2.2 | 正确答案准确性 | ≥ 90% | ⚠️ 依赖 LLM 质量 |
| AC2.3 | 干扰项合理性 | ≥ 75% | ✅ assistant.md 约束：每个干扰项对应迷思概念 |
| AC2.4 | 知识点覆盖度 | 每个 chunk 至少 1 题 | ✅ prompt 要求 |
| AC2.5 | 难度分布 | 记忆:理解:应用:分析 ≈ 3:4:2:1 | ✅ prompt 约束 |
| AC2.6 | JSON 解析成功率 | ≥ 95%（4 层防御） | ✅ |
| AC2.7 | question_count 参数生效 | 传入 N 生成 N 道 | ✅ v0.3.0 新增 |

**已知限制：**
- 前端 6 种题型中 4 种渲染为文本框（true_false/fill_blank/matching 无专用 UI）
- chunks 超过 20 个时均匀采样，部分知识点可能未被出题
- `question_count` 参数仅在 v0.3.0 生效（之前被丢弃）

### F4 迷思概念诊断 ✅ 已实现

**用户故事：**
> 作为一名学生，我希望系统不仅仅判对错，还能分析我错在哪里——是概念混淆、计算错误还是粗心——这样我才能针对性地弥补。

**实现文件：**
- 流水线：`orchestrator_phases.py` → `_evaluating_phase()` → Evaluator 角色

- 前端：`ResultsPage.tsx` → 迷思概念诊断区（severity 标签颜色 + remediation_hint）

**7 种错误类别：**

| 类别 | 标识 | 示例 |
|------|------|------|
| 概念混淆 | `conceptual` | 动量与动能混淆 |
| 计算错误 | `calculation` | 符号错误；数值代入错误 |
| 应用不当 | `application` | 公式选错；条件判断遗漏 |
| 逻辑错误 | `logic` | 推理链条断裂；因果关系颠倒 |
| 粗心 | `careless` | 漏看条件；抄错数字 |
| 符号书写 | `notation` | 单位遗漏 |
| 不完整 | `incomplete` | 只写结论无过程 |

### F5 SM-2 排期计划 ✅ 已实现

**用户故事：**
> 作为一名学生，我希望系统根据我的作答表现自动生成每日复习计划，薄弱点多排、已掌握的少排，像真人私教一样帮我分配精力。

**实现文件：**
- 流水线：`orchestrator_phases.py` → `_planning_phase()`（Tutor 角色）+ `_persist_mastery_records()`
- Prompt：`orchestrator_prompts.py` → `_build_planning_prompt()`
- 路由：`routes/dashboard.py` → `GET /plan/today` + `POST /plan/items/{id}/toggle`
- 前端：`PlanPage.tsx`（今日复习清单 + checkbox 完成切换）
- 数据库：`mastery_records` + `study_plans` + `review_items` 表

**SM-2 参数：**

| 参数 | 初始值 | 范围 | 说明 |
|------|--------|------|------|
| EF | 2.5 | [1.3, +∞) | 难易度因子 |
| 首次通过间隔 | 1 天 | — | 连续正确 1 次 |
| 二次通过间隔 | 6 天 | — | 连续正确 2 次 |
| 后续间隔 | `interval × EF` | — | 第 3 次及以后 |
| 失败时 | interval=1, rep=0 | — | 回到初始 |
| 掌握度更新 | EMA α=0.3, decay=0.4 | — | 平滑更新 |

**验收标准：**

| ID | 标准 | 量化指标 | 状态 |
|----|------|---------|:---:|
| AC5.1 | 排期依据 | SM-2 算法完整实现 | ✅ |
| AC5.2 | 优先级公式 | `(1 - mastery) × log(1 + overdue_days)` | ✅ |
| AC5.3 | 每日复习量上限 | ≤ 2 小时 | ✅ |

## 附录 A · 词汇表

| 术语 | 英文 | 定义 |
|------|------|------|
| 检索增强生成 | RAG | 先从知识库检索相关内容，再提交给 LLM 生成回答 |
| 嵌入向量 | Embedding | 文本在高维向量空间的数值表示 |
| 间隔重复 | Spaced Repetition | 根据遗忘曲线在最佳时机安排复习 |
| 难易度因子 | Ease Factor (EF) | SM-2 算法核心参数 |
| 指数移动平均 | EMA | 加权平均方法，掌握度平滑（α=0.3） |
| 认知孪生 | Cognitive Twin | 每个知识点的数学化掌握度模型 |
| 迷思概念 | Misconception | 学生的系统性错误认知 |
| 苏格拉底式教学 | Socratic Method | 通过提问引导学生自主发现答案 |
