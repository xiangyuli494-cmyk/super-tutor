## 第 1 章 · 项目概述

### 1.1 产品愿景

超级私教是一个**基于 LLM 的深度学习辅助工具**。用户选择课程类型、上传学习资料后，系统自动提取知识点，通过诊断评估确定用户水平，制定个性化学习计划，并支持多题型练习、自动判题、错题整理与苏格拉底式追问。

### 1.2 一句话定位

> 上传资料 → 提取知识点 → 定位薄弱项 → 精准练习 → 错题追问 —— 一个完整的自学闭环。

### 1.3 核心设计原则

1. **简单优先**：前端用 Streamlit 单页应用，一个 `streamlit run` 就能启动全部功能
2. **知识点为中心**：数据库的核心实体是知识点，所有功能围绕它展开
3. **无状态机**：去掉庞大的 Orchestrator 状态机，改用「页面 + 按钮」驱动用户流程
4. **前后知识点联动**：利用知识点间的前驱/后继关系提高评估和出题的准确性
5. **错题即入口**：错题本不只是记录，更是苏格拉底式追问学习的起点

### 1.4 目标用户

- **大学生**：专业课教材 PDF 多，备考压力大，需要系统化复习工具
- **自学者**：想系统学一门新技术/新领域，需要结构化理解和测验

### 1.5 用户画像

| 用户 | 典型场景 | 痛点 | 超私如何解决 |
|------|---------|------|------------|
| 大学生 | 期末周 5 门课，3 本厚教材 | "来不及看，不知道从哪开始" | 上传 PDF → 自动提取知识点 → 诊断薄弱点 → 专攻练习 |
| 自学者 | 想系统学一门新技术 | "资源太多，不知道重点在哪" | 任意文档上传，自动提炼知识点结构 |

## 第 2 章 · 功能需求

### 整体流程

```
选择课程类型 → 上传学习资料 → 解析提取知识点 → 诊断评估
    → 制定学习计划 → 多题型练习 → 自动判题 → 错题本 → 苏格拉底追问
```

### F1 课程类型选择

**描述：**
用户首先选择课程的类型（如：计算机科学、数学、物理、化学、英语、自定义等），系统据此调整解析策略和出题风格。

**功能点：**
- 预设课程类型模板（计算机、数学、物理、化学、英语等）
- 每种类型影响：知识点拆分粒度、出题题型偏好、难度锚定
- 支持"自定义"类型，用户自行命名

**验收标准：**

| ID | 标准 | 量化指标 |
|----|------|---------|
| AC1.1 | 课程类型选择后进入对应上传页 | 点击即可切换 |
| AC1.2 | 不同课程类型有不同默认配置 | 至少 3 种类型有差异化配置 |

---

### F2 资料上传与知识点解析

**描述：**
用户上传 PDF（或粘贴文本），系统提取纯文本后调用 LLM 按语义边界将内容拆分为知识点，存入数据库。

**功能点：**
- 支持 PDF 上传（PyMuPDF 提取文本）
- 支持文本直接粘贴
- LLM 按知识点语义边界拆分（每个知识点 chunk 有明确的前驱/后继关系）
- 每个知识点包含：标题、内容摘要、关键词、难度标记
- **前后知识点关联**：识别知识点间的依赖关系（前驱知识点/后继知识点），提高后续评估和出题的准确性

**知识点结构：**

```
知识点 (KnowledgePoint)
├── 标题          # e.g. "牛顿第二定律"
├── 内容摘要       # 200 字以内的核心提炼
├── 原文片段       # 教材中的原始表述
├── 关键词         # ["力", "加速度", "质量", "F=ma"]
├── 难度           # beginner / easy / medium / hard
├── 前驱知识点     # [KP_id_1, KP_id_2] — 学习本知识点前应先掌握的知识点
├── 后继知识点     # [KP_id_3] — 本知识点为哪些知识点打基础
└── 课程类型       # e.g. "物理"
```

**验收标准：**

| ID | 标准 | 量化指标 |
|----|------|---------|
| AC2.1 | PDF 文本提取完整率 | ≥ 95% |
| AC2.2 | 知识点切分语义完整率 | ≥ 85% |
| AC2.3 | 前后知识点关系识别准确率 | ≥ 75% |
| AC2.4 | 50 页 PDF 解析耗时 | ≤ 45 秒 |

---

### F3 诊断评估

**描述：**
知识点提取完成后，系统生成一份诊断评估测验（约 10-20 题，覆盖所有知识点），评估用户当前水平，为后续学习计划提供依据。

**功能点：**
- 覆盖所有已提取的知识点，每个知识点至少 1 题
- 从简单的知识点开始，逐步递进
- 利用前后知识点关系：若某知识点答错，自动标记其所有后继知识点为"高风险"
- 评估结果输出：每个知识点的掌握度初值（0-1）
- 生成诊断报告：薄弱知识点列表、强项列表、建议学习顺序

**前后知识点联动规则：**

```text
规则 1: 知识点 A 是 B 的前驱 → 若 A 未掌握（得分 < 0.5），则 B 的评估可信度降低
规则 2: 若 B 答对但 A 答错 → 标记为"需复查"（可能是猜测或知识迁移不完整）
规则 3: 连续 3 个后继知识点答错 → 判定前驱知识点需要重新学习
```

**验收标准：**

| ID | 标准 | 量化指标 |
|----|------|---------|
| AC3.1 | 诊断评估覆盖所有知识点 | 100% 覆盖 |
| AC3.2 | 前后知识点联动规则生效 | 3 条规则均可触发 |
| AC3.3 | 评估报告可读性 | 用户能清晰看到强弱项 |

---

### F4 学习计划制定

**描述：**
基于诊断评估结果和知识点前后依赖关系，生成个性化的学习计划。薄弱点多排、已掌握的点少排，按照知识点依赖拓扑排序安排学习顺序。

**功能点：**
- 按知识点依赖拓扑排序（前驱 → 后继）
- 薄弱知识点优先、高频安排
- 每日学习量控制在合理范围
- 学习计划可查看、可调整

**优先级公式：**

```text
优先级 = (1 - 掌握度) × (1 + 后继知识点数量 / 总知识点数量)
```

> 未掌握且是多个后续知识点的基础 → 最高优先级

**验收标准：**

| ID | 标准 | 量化指标 |
|----|------|---------|
| AC4.1 | 学习计划按依赖顺序排列 | 前驱知识点始终在后继之前 |
| AC4.2 | 薄弱点优先 | 掌握度 < 0.5 的知识点在前 50% |
| AC4.3 | 每日学习量提示 | 给出预估学习时间 |

---

### F5 多题型出题与答题

**描述：**
用户可针对特定知识点（或全部知识点）生成练习题。系统支持多种题型，每次出题时注入前后知识点上下文以提高题目质量。

**支持的 6 种题型（`QuestionType` 枚举）：**

| 题型 | type 值 | 占比 | 前端渲染 |
|------|---------|:---:|---------|
| 选择题 | `multiple_choice` | ~40% | `st.radio()` 单选按钮 |
| 判断题 | `true_false` | ~15% | `st.radio(["对", "错"])` |
| 填空题 | `fill_in_blank` | ~15% | `st.text_input()` 填空输入框 |
| 简答题 | `short_answer` | ~15% | `st.text_area()` 多行文本框 |
| 论述题 | `essay` | ~10% | `st.text_area()` 大段文本框 |
| 编程题 | `coding` | ~5% | `st.text_area()` 代码输入框 |

> 6 种题型覆盖布鲁姆认知层次：记忆(选择/判断) → 理解(填空) → 应用(简答) → 分析/综合(论述/编程)。

**前后知识点出题策略：**

```text
1. 出题时向 LLM 同时提供「目标知识点 + 前驱知识点摘要」
   → 帮助 LLM 生成更有层次感、更能测试理解深度的题目
2. 干扰项设计时引用相关前驱知识点的常见迷思概念
   → 提高干扰项质量
3. 难度递进：先考前驱知识点的基础题 → 再考目标知识点的综合题
```

**验收标准：**

| ID | 标准 | 量化指标 |
|----|------|---------|
| AC5.1 | 题干可理解率 | ≥ 95% |
| AC5.2 | 正确答案准确性 | ≥ 90% |
| AC5.3 | 干扰项合理性 | ≥ 75% |
| AC5.4 | 前后知识点上下文注入 | 每次出题 Prompt 携带前驱知识点摘要 |

---

### F6 自动判题与反馈

**描述：**
用户提交答案后，系统自动判断对错并给出详细解析。客观题（选择题、判断题）程序化判题，填空题用 LLM 辅助判题。

**功能点：**
- 选择题/判断题：程序化比对，100% 准确
- 填空题：LLM 语义匹配 + 关键词比对
- 简答题/论述题/编程题：LLM 按关键点覆盖度评分
- 每题给出详细解析，错题自动收录到错题本
- 更新对应知识点的掌握度

**验收标准：**

| ID | 标准 | 量化指标 |
|----|------|---------|
| AC6.1 | 选择题/判断题批改准确率 | 100%（程序化） |
| AC6.2 | 填空题批改准确率 | ≥ 85% |
| AC6.3 | 简答题/论述题/编程题关键点匹配率 | ≥ 75% |
| AC6.4 | 批改响应时间（10 题） | ≤ 5 秒 |

---

### F7 错题本

**描述：**
系统自动收录所有错题，按知识点分类展示。用户可以回顾错题、查看解析、重新作答。

**功能点：**
- 自动收录所有答错的题目
- 按知识点分组展示
- 显示：题目、用户答案、正确答案、解析、犯错次数、收录时间
- 支持筛选：按知识点、按时间范围
- 支持重新作答
- **苏格拉底入口**：每个错题关联的知识点可发起苏格拉底式追问学习

**错题本数据结构：**

```
错题记录 (WrongQuestion)
├── 题目内容
├── 用户答案
├── 正确答案
├── 解析
├── 关联知识点 ID
├── 犯错次数
├── 最近犯错时间
└── 是否已订正
```

**验收标准：**

| ID | 标准 | 量化指标 |
|----|------|---------|
| AC7.1 | 错题自动收录率 | 100% |
| AC7.2 | 按知识点分组展示 | 每个知识点一个分组 |
| AC7.3 | 错题本入口可见 | 侧边栏直接进入 |

---

### F8 苏格拉底式追问

**描述：**
用户在错题本中，可以对某道错题关联的**知识点**发起苏格拉底式对话。系统不直接给出答案，而是通过层层递进的问题引导用户自主发现正确理解。

**功能点：**
- **入口**：错题本中每个知识点旁有"苏格拉底追问"按钮
- **三层引导**：
  - L1（笼统引导）："想想这个公式中的每个符号代表什么？"
  - L2（方向提示）："如果质量不变，力增大一倍，加速度会怎么变？"
  - L3（接近答案）："F=10N，m=2kg，代入公式 a=F/m 得到什么？"
- 每次追问记录对话历史
- 用户可以随时选择"我知道了"或"显示答案"退出追问模式

**对话模式：**

```text
系统: 你回顾一下，牛顿第二定律的公式是什么？        (L1)
用户: F=ma
系统: 对。那么题目中已知 F=10N, m=2kg，应该怎么求 a？ (L2)
用户: a = 10/2 = 5
系统: 很好！那你现在能理解为什么答案是 5 m/s² 了吗？  (L3)
```

**验收标准：**

| ID | 标准 | 量化指标 |
|----|------|---------|
| AC8.1 | 苏格拉底对话至少 3 轮递进 | L1 → L2 → L3 |
| AC8.2 | 对话历史可追溯 | 显示完整追问记录 |
| AC8.3 | 用户可随时退出 | "显示答案"按钮可见 |

## 第 3 章 · 非功能需求

### 3.1 性能

| 指标 | 目标 |
|------|------|
| PDF 解析（50 页） | ≤ 45 s |
| 单次出题（10 题） | ≤ 25 s |
| 批量批改（10 题） | ≤ 5 s |
| Streamlit 页面切换 | ≤ 1 s |

### 3.2 可用性

- 所有 LLM 调用有 loading 状态（spinner + 文案）
- 超时提示：120s 超时 → 友好重试提示
- 关键操作提供重试按钮
- Streamlit 原生的简洁界面，无需前端技术栈

### 3.3 安全性

- API Key 通过环境变量或 Streamlit secrets 注入
- LLM 调用通过后端代理，前端不持有 API Key
- 题目正确答案在渲染前剔除

### 3.4 技术栈精简

| 层 | v4.0（旧） | v5.0（新） | 变化 |
|----|-----------|-----------|------|
| 前端 | React + Vite + Tailwind + Zustand | **Streamlit** | 去掉 4 个前端技术栈 |
| 后端 | FastAPI + Uvicorn | FastAPI + Uvicorn | 保留 |
| 编排 | Orchestrator 状态机（~2000 行） | **无** — 页面按钮驱动 | 去掉整个状态机 |
| 数据库 | SQLite 14 表 | **SQLite 6 表** | 大幅精简 |
| LLM | DeepSeek API | DeepSeek API | 保留 |
| AI 角色 | Tutor/Assistant/Evaluator (3 角色) | **单一 LLM 调用** | 去掉角色系统 |

## 附录 A · 词汇表

| 术语 | 英文 | 定义 |
|------|------|------|
| 知识点 | Knowledge Point (KP) | 系统的最小学习单元，由一个不可再分的概念/公式/定理构成 |
| 前驱知识点 | Prerequisite KP | 学习目标知识点前必须先掌握的知识点 |
| 后继知识点 | Successor KP | 以目标知识点为基础才能学习的知识点 |
| 拓扑排序 | Topological Sort | 按知识点依赖关系排列学习顺序的算法 |
| 苏格拉底式教学 | Socratic Method | 通过提问引导学生自主发现答案，而非直接告知 |
| RAG | Retrieval-Augmented Generation | 检索增强生成 |
